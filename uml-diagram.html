Authentication Sequence Diagram

Login flow with CAPTCHA, TOTP 2FA, account lockout, and session creation.

sequenceDiagram
    actor U as User
    participant F as Frontend
    participant S as Express Server
    participant DB as Database
    participant E as Email / OAuth

    rect rgb(22, 27, 34)
    Note over U,E: Registration Flow
    U->>F: Fill register form
    F->>S: GET /api/auth/captcha
    S-->>F: CAPTCHA SVG + session ID
    U->>F: Solve CAPTCHA + submit
    F->>S: POST /api/auth/register
    S->>S: Validate inputs + honeypot
    S->>S: Check rate limit (3/hr, 10/day)
    S->>DB: INSERT user + user_settings
    S->>DB: INSERT email_verification_token
    S->>E: Send verification email
    S-->>F: 201 Created
    end

    rect rgb(22, 27, 34)
    Note over U,E: Login Flow (with 2FA)
    U->>F: Enter email + password
    F->>S: POST /api/auth/login
    S->>S: Check rate limit (5/min)
    S->>DB: SELECT user by email
    alt Account locked
        S-->>F: 423 Locked (locked_until)
    end
    S->>S: bcrypt.compare(password)
    alt Password wrong
        S->>DB: INCREMENT failed_login_attempts
        alt Exceeded threshold
            S->>DB: SET locked_until
        end
        S-->>F: 401 + captchaRequired flag
    end
    alt TOTP enabled
        S-->>F: 200 + totpRequired: true
        U->>F: Enter 6-digit TOTP code
        F->>S: POST /api/auth/login (with totp)
        S->>S: Verify TOTP (HMAC-SHA1, 30s step)
    end
    S->>DB: UPDATE last_login, login_count
    S->>S: Create session (set cookie)
    S-->>F: 200 + user data
    end

    rect rgb(22, 27, 34)
    Note over U,E: OAuth Flow (Google/GitHub)
    U->>F: Click OAuth button
    F->>S: GET /api/oauth/google
    S->>S: Generate CSRF state token
    S-->>F: 302 Redirect to Google
    F->>E: Google consent screen
    E-->>F: Redirect with auth code
    F->>S: GET /api/oauth/google/callback
    S->>E: Exchange code for token
    S->>E: Fetch user profile
    S->>DB: Find or create user
    S->>S: Regenerate session ID
    S-->>F: 302 Redirect /#oauth-success
    end

Data & Caching Flow

Service Worker caching strategies and offline-first data flow for price data.

sequenceDiagram
    actor U as User
    participant App as Frontend App
    participant SW as Service Worker
    participant C as Cache API
    participant API as Express API
    participant Ext as EIA / Yahoo

    rect rgb(22, 27, 34)
    Note over U,Ext: Online - Price Data Fetch
    U->>App: Load dashboard
    App->>SW: fetch(/api/...) or Yahoo URL
    SW->>SW: Route matching
    alt API request (/api/*)
        SW->>API: Network-first strategy
        API-->>SW: JSON response
        SW->>C: Cache response (oil-runtime-v6)
        SW-->>App: Return data
    else External data (EIA/Yahoo)
        alt Fast connection (4G+)
            SW->>Ext: Network-first
            Ext-->>SW: Price data
            SW->>C: Cache (oil-data-v6, max 80)
        else Slow connection (2G/3G)
            SW->>C: Cache-first lookup
            alt Cache hit
                C-->>SW: Cached data
                SW-->>App: Stale data + X-From-Cache
                App->>App: Show stale badge
            else Cache miss
                SW->>Ext: Fetch with adaptive timeout
            end
        end
    end
    App->>App: NetInfo adjusts resolution
    App->>App: drawSparkline (downsample if slow)
    end

    rect rgb(22, 27, 34)
    Note over U,Ext: Offline - Analytics Queue
    U->>App: Navigate / interact
    App->>App: Analytics.send(event)
    alt Online
        App->>API: sendBeacon POST
    else Offline
        App->>App: Enqueue to localStorage
        App->>App: Update pending count badge
        Note over App: On reconnect or SW sync event
        App->>App: flushQueue()
        loop Each queued event
            App->>API: fetch with keepalive
        end
    end
    end

    rect rgb(22, 27, 34)
    Note over U,Ext: Static Assets
    U->>App: Request page
    App->>SW: fetch(index.html)
    SW->>C: Stale-while-revalidate
    C-->>SW: Cached shell (oil-shell-v6)
    SW-->>App: Serve from cache
    SW->>API: Background revalidate
    end
