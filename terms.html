2.1 Data Protection and Backups

We maintain a comprehensive automated SQLite backup infrastructure to protect your data against loss:

  • Backup retention policy: Hourly snapshots for point-in-time recovery, 7 daily backups, 4 weekly backups, and 12 monthly backups. Older backups are automatically rotated and securely deleted
  • Point-in-time recovery: Using periodic snapshots combined with WAL archiving, the Service can restore the database to any specific moment in time, minimising data loss beyond the last scheduled backup
  • Backup integrity verification: Every backup undergoes SHA-256 checksum validation and periodic automated restore tests against an isolated environment to confirm recoverability, schema integrity, and data consistency
  • One-click restore: Authorised administrators can initiate a restore from any available recovery point (hourly, daily, weekly, monthly) via the admin panel, with automatic pre-restore backup creation to prevent data loss during the restore operation
  • WAL checkpoint management: Active checkpoint management ensures WAL files remain compact, all data is durably written, and backup snapshots capture a fully consistent database state
  • Data migration tooling: The Service includes tooling to export data from SQLite to PostgreSQL or other relational databases if the platform needs to scale. Migrations include automated validation of row counts, data types, and referential integrity

All backups are stored on our server infrastructure, encrypted at rest, and not shared with third parties. Upon account deletion, your personal data is purged from all backup copies within the next hourly backup cycle. For complete technical details, see our Privacy Policy, Sections 6.4–6.7.

2.2 Graceful Shutdown and Data Integrity

The Service implements a graceful shutdown procedure to protect your data during server maintenance, updates, and restarts:

  • All in-flight requests are completed before the server stops accepting new connections
  • The SQLite database connection is closed cleanly, ensuring all pending writes are flushed and committed without corruption
  • Active user sessions are preserved across restarts — you will not be forcibly logged out during routine maintenance
  • The process handles SIGTERM and SIGINT signals for orderly shutdown rather than abrupt termination

While we take every measure to maintain continuous service, scheduled maintenance may result in brief periods of unavailability. No data loss, corrupted records, or interrupted transactions will occur as a result of these maintenance windows. For full technical details, see our Privacy Policy, Section 6.6.
